How to Install

Follow these simple steps to install the Student Scheduler App on your Android device.

1

Download the APK

Click the "Download APK" button on this webpage to start downloading the Student Scheduler App to your Android device.

2

Confirm Download

Your browser will prompt you to confirm the download. Tap "Download" or "OK" to save the APK file to your device's Downloads folder.

3

Install the App

Open the downloaded APK file from your notifications or Downloads folder. Your device will scan the file for security, then show the install screen. Tap "Install" to complete the installation.

info

Important Note

You may need to enable "Install from unknown sources" in your Android settings to install APK files. This is normal for apps not downloaded from the Google Play Store.

Database Design & Entity Models

Entity Relationships

User (1:N)
Id, Username, PasswordHash, Salt, CreatedAt, IsActive
Term (1:N with Courses)
Id, Name, StartDate, EndDate, UserId
Course (1:N with Assessments)
Id, Name, StartDate, EndDate, Status, InstructorInfo, Notes, TermId, UserId
Assessment
Id, Name, StartDate, EndDate, Type, CourseId, UserId

SQLite Implementation

  • ORM: sqlite-net-pcl 1.9.172
  • Attributes: [Table], [PrimaryKey], [AutoIncrement], [Ignore]
  • Foreign Keys: Enforced relationships with cascade operations
  • Data Isolation: UserId filtering on all queries
  • Async Operations: All database calls use async/await
  • Connection Management: SQLiteAsyncConnection pattern
  • Indexing: Optimized queries with proper indexing

Security Architecture

Authentication Security

  • Password Hashing: SHA256 with salt using System.Security.Cryptography
  • Salt Generation: Unique salt per user for enhanced security
  • Session Management: Secure user context with logout functionality
  • Input Validation: Comprehensive validation with ArgumentNullException
  • Data Sanitization: SQL injection prevention through ORM

Data Protection

  • User Isolation: Complete data segregation via UserId filtering
  • Local Storage: SQLite database stored in app's private directory
  • No Cloud Dependencies: All data remains on device
  • Error Handling: Graceful exception handling with logging
  • Debug Logging: Microsoft.Extensions.Logging for development
